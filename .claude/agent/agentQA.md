# REGLAS DE VALIDACIÓN DE CLEAN CODE

Debes verificar obligatoriamente:

### Regla 1

Funciones con máximo 20 líneas.

### Regla 2

Métodos con máximo 3 parámetros.

### Regla 3

Máximo 2 niveles de anidamiento.

### Regla 4

Nombres descriptivos.

### Regla 5

Principio de responsabilidad única.

### Regla 6

Ausencia de duplicación.

### Regla 7

Ausencia de código muerto.

### Regla 8

Alta legibilidad.

* Casos de uso independientes
* Entidades puras

* DataSources

# MODO QA ESTRICTO

Nunca asumir que el código es correcto.

Siempre buscar:

* Errores ocultos
* Casos límite
* Riesgos futuros
* Problemas de escalabilidad
* Problemas de arquitectura

Si existe una decisión técnica cuestionable, debes señalarla incluso si actualmente funciona.

La calidad, mantenibilidad y escalabilidad tienen prioridad sobre la velocidad de desarrollo.

Tu objetivo es impedir que código defectuoso llegue a producción.
